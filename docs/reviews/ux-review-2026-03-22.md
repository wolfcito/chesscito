## Critical (must fix)

### C1 — Navigation pattern chaos across secondary screens
**Files:** `leaderboard/page.tsx`, `trophies/page.tsx`, `victory/[id]/page.tsx`, `tx-lab/page.tsx`
Three incompatible back-navigation patterns:
| Screen | Pattern | Back destination |
|---|---|---|
| Leaderboard | `<AppShell cta={...}>` managed by shell | `/` |
| Trophies | Manual inline `<Link>` + ArrowLeft icon in hero | `/` |
| Victory | Plain text link "Back to Hub" `text-xs text-white/30` | `/` |
| TX Lab | No back button at all | — |
**Fix:** Pick one pattern (ideally `AppShell` or a shared `<BackButton>`) and apply across all secondary screens.

### C2 — `DIFFICULTY_LABEL` in victory page duplicates editorial + possibly wrong mapping
**File:** `victory/[id]/page.tsx:7`
```tsx
const DIFFICULTY_LABEL: Record<number, string> = { 1: "Easy", 2: "Medium", 3: "Hard" };
```
These labels already exist in `editorial.ts` as `ARENA_COPY.difficulty`. Worse, the keys `1/2/3` may not match js-chess-engine levels `1/3/5` — "Medium" and "Hard" may show incorrect labels.
**Fix:** Import from `editorial.ts` and verify the difficulty-to-level mapping.

### C3 — `purchase-confirm-sheet.tsx` exposes raw chainId and contract address to users
**File:** `purchase-confirm-sheet.tsx:65,68`
- Line 65: `Network: {chainId ?? "n/a"}` — shows raw number `42220`
- Line 68: `Shop: {shopAddress ?? "missing"}` — shows full hex address + developer-language fallback
**Fix:** Map chainId to human-readable name ("Celo"), truncate or hide contract address.

### C4 — `tx-lab/page.tsx` uses `max-w-5xl` (1024px) breaking mobile-first constraint
**File:** `tx-lab/page.tsx:256`
The app's design system is `--app-max-width: 390px`. This page uses `max-w-5xl` = 1024px.
**Fix:** Apply `max-w-[var(--app-max-width)]` or mark this route as developer-only/hidden.

### C5 — `victory/[id]/page.tsx` missing app max-width container
**File:** `victory/[id]/page.tsx:92`
Root `<main>` has no `max-w-[var(--app-max-width)]`. The card is constrained to `max-w-[340px]` but the background expands unbounded.
**Fix:** Add `mx-auto max-w-[var(--app-max-width)]` to the container.

## Major (should fix)

### M1 — No shared `<Button>` component used in game UI
**48 raw `<button>` instances** across 25 files. A `<Button>` component exists in `components/ui/button.tsx` with CVA variants, but the entire Arena, Play Hub overlays, and Victory flows use raw `<button>` with one-off class strings.
**Key inconsistencies from this:**
- `rounded-xl` vs `rounded-2xl` on primary CTAs (no clear rule)
- `disabled:opacity-40` vs `disabled:opacity-50` vs `disabled:opacity-70` (3 different values)
- `bg-cyan-600` (flat) vs `from-cyan-500 to-cyan-400` (gradient) vs `from-cyan-500 to-teal-400` (different gradient) for the same "primary action" intent
**Files:** `contextual-action-slot.tsx`, `result-overlay.tsx`, `badge-sheet.tsx`, `shop-sheet.tsx`, `mission-briefing.tsx`, `arena-end-state.tsx`, `difficulty-selector.tsx`, all victory components

### M2 — `#0a1424` frosted panel base repeated 8+ times with no CSS variable
**Files:** `arena-end-state.tsx:117`, `difficulty-selector.tsx:23`, `promotion-overlay.tsx:35`, `victory-celebration.tsx:76`, `victory-claiming.tsx:37`, `victory-claim-success.tsx:96`, `victory-claim-error.tsx:49`, `victory/[id]/page.tsx:93`, `trophies/page.tsx:92`
The frosted panel token `bg-[#0a1424]/92` is copy-pasted across 9 files. A single brand color change requires 9 manual edits.
**Fix:** Extract to a CSS variable like `--surface-frosted` or a Tailwind utility class.

### M3 — Overlay scrim opacity inconsistency
**9 files** use `bg-black` with varying opacities for the same overlay backdrop pattern:
- `bg-black/50` — `sheet.tsx`
- `bg-black/60` — `arena-end-state.tsx`
- `bg-black/65` — `promotion-overlay.tsx`
- `bg-black/70` — `result-overlay.tsx`, `victory-celebration.tsx`, `victory-claiming.tsx`, `victory-claim-success.tsx`, `victory-claim-error.tsx`
**Fix:** Standardize on one value (likely `bg-black/70`) or create `--overlay-scrim`.

### M4 — Touch targets under 44px minimum
| Component | File:Line | Size | Issue |
|---|---|---|---|
| Arena back button | `arena-hud.tsx:49-56` | 32×32px (`h-8 w-8`) | Below 44px |
| Arena resign button | `arena-hud.tsx:73-93` | 32×32px resting state | Below 44px |
| Trophies back button | `trophies/page.tsx:99` | 32×32px (`h-8 w-8`) | Below 44px |
| Victory "Back to Hub" | `victory/[id]/page.tsx:126-131` | `text-xs` link, no padding | Minimal target |
| Exercise drawer rows | `exercise-drawer.tsx:93-143` | ~40px (`py-2.5`) | Marginally under |
| Arena end-state secondary btn | `arena-end-state.tsx:139-145` | ~40px (`py-2.5`) | Marginally under |

### M5 — `text-red-*` vs `text-rose-*` mixed for error states
- `text-red-400` used in: `leaderboard-sheet.tsx:74`, `shop-sheet.tsx:62`, `trophy-list.tsx:43,48`, `trophy-card.tsx:11`
- `text-rose-300/400` used in: `result-overlay.tsx`, `tx-feedback-card.tsx`, `mission-panel.tsx`, `arena-hud.tsx`, `arena-end-state.tsx`
These are visually different Tailwind palettes. Error states should use one family consistently.
**Fix:** Standardize on `rose-*` (matches destructive semantic) or extract to `--color-error`.

### M6 — Mission briefing has no close/escape button
**File:** `mission-briefing.tsx`
The only way to dismiss is the "Play Now" CTA or the Arena link. No X button, no swipe-down, no tap-outside-to-close. Users who've already played cannot easily skip.

### M7 — Safe area bottom padding missing on non-AppShell screens
**Files:** `trophies/page.tsx:115` (`pb-8` only), `victory/[id]/page.tsx:93` (`pb-8` only)
No `env(safe-area-inset-bottom)` handling. Content may be cut off under the home indicator on notched iPhones / MiniPay WebView.

### M8 — Arena back button resets game without confirmation
**File:** `arena/page.tsx:295`
`onBack={game.reset}` silently destroys game state. Resign has double-tap confirmation, but back does not. Same visual button (frosted circle + ArrowLeft), different destructive behavior.

### M9 — Inline copy strings bypass editorial.ts
**Files with inline user-facing strings that should reference editorial constants:**
| File | Line | String |
|---|---|---|
| `victory/[id]/page.tsx` | 114 | `"Can you beat this?"` |
| `victory/[id]/page.tsx` | 122 | `"Accept Challenge"` |
| `victory/[id]/page.tsx` | 129 | `"Back to Hub"` |
| `mission-briefing.tsx` | 25 | `Move your ${pieceType}...` (template) |
| `exercise-drawer.tsx` | 90 | `Exercise ${index + 1}` (fallback) |
| `page.tsx` | 578,582 | Technical error messages reaching UI |

### M10 — Card/heading styles diverge across screens
Three radically different card styles with no shared abstraction:
- **Leaderboard:** `rounded-2xl border border-slate-200` (light theme)
- **Trophies:** `bg-[#121c2f]` (custom dark hex)
- **Victory:** `rounded-3xl border-white/[0.08] bg-[#0a1424]/92 backdrop-blur-2xl` (frosted dark)

Heading typography also diverges: `text-xl font-bold text-white` vs `text-2xl font-bold text-emerald-300` vs `text-2xl font-semibold text-slate-950`.

### M11 — Tailwind config hardcodes primary/secondary colors as hex
**File:** `tailwind.config.ts`
`primary.DEFAULT: "#07955F"` and `secondary.DEFAULT: "#2A2C34"` are hardcoded hex values instead of referencing `hsl(var(--primary))`. This means changing CSS variables won't affect Tailwind utilities.

### Top 3 systemic issues

1. **No design token layer between Tailwind colors and components.** CSS variables exist (`--primary`, etc.) but are only used by shadcn/ui primitives. All custom game UI hardcodes Tailwind color classes. `#0a1424` alone appears in 9 files.

2. **48 raw `<button>` elements** with inconsistent styling (3 different disabled opacities, 2 different border-radii, 3 different gradient variants for primary CTAs). The existing `<Button>` component is unused outside tx-lab.

3. **Navigation pattern fragmentation.** Secondary screens use 3 different back-button implementations. No shared navigation component or convention.
