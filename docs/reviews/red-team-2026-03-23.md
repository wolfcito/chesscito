## CRITICAL — Backend Security

### C1. No Game Session Verification — Arbitrary NFT Minting
- **Route**: `sign-victory`, `sign-score`, `sign-badge`
- **File**: `app/api/sign-victory/route.ts:20-32`, `sign-score/route.ts:20-32`, `sign-badge/route.ts:20-23`
- **Description**: Server signs EIP-712 data based entirely on client-supplied `difficulty`, `totalMoves`, `timeMs`. No server-side game session exists. An attacker can mint fake Victory NFTs with arbitrary stats.
- **Attack**: `curl -X POST /api/sign-victory -d '{"player":"0xAttacker","difficulty":3,"totalMoves":1,"timeMs":1}'`
- **Impact**: Integrity of ALL minted NFTs compromised. Becomes fund-theft vector when v2 prize pool launches.
- **Known**: Documented as v1 accepted risk in project memory. Must fix before prize pool.

### C2. Coach Credit Race Condition — Non-Atomic Seeding
- **Route**: `coach/analyze`, `coach/credits`
- **File**: `app/api/coach/analyze/route.ts:52-58`, `app/api/coach/credits/route.ts:15-20`
- **Description**: Free credit seeding uses check-then-set (not atomic). Two concurrent requests can both seed. Worse, `/credits` and `/analyze` seed independently — calling `/credits` after spending credits can reset balance to 3 if seeded key was evicted.
- **Impact**: Unlimited free LLM analyses, costing real API money.
- **Fix**: Use `SETNX` for seeded flag + credits atomically. Consolidate seeding into one function.

## HIGH — Backend Security

### H1. Missing `await` on `enforceRateLimit` in sign-score and sign-badge
- **File**: `app/api/sign-score/route.ts:27`, `app/api/sign-badge/route.ts:22`
- **Description**: Rate limit check fires async but is never awaited. Signing proceeds regardless.
- **Impact**: Rate limiting on score/badge signing is non-functional.
- **Fix**: Add `await` (trivial one-word fix).

### H2. Verify-Purchase: Event Signature Not Checked
- **File**: `app/api/coach/verify-purchase/route.ts:41-57`
- **Description**: Log parsing matches by contract address only, doesn't verify `topics[0]` event signature hash. Any event with 3+ topics would be misinterpreted as a purchase.
- **Fix**: Check `log.topics[0]` against expected `ShopPurchase` event hash.

### H3. No Wallet Address Validation on Coach/Games Routes
- **Files**: All coach routes + games route
- **Description**: `walletAddress` only gets `.toLowerCase()`, never validated as Ethereum address. Arbitrary strings become Redis keys.
- **Fix**: Use `isAddress()` from viem (already used in my-victories route).

### H4. Redis Key Injection via Unsanitized gameId
- **File**: `app/api/games/route.ts:13`, `app/api/coach/analyze/route.ts:25`
- **Description**: `gameId` used directly in Redis keys with no format validation.
- **Fix**: Validate as UUID format (`/^[0-9a-f-]{36}$/i`).

### H5. Coach Job IDOR — Any User Can Read Any Job
- **File**: `app/api/coach/job/[id]/route.ts:8-16`
- **Description**: Job polling returns results for ANY job ID without ownership verification.
- **Fix**: Store wallet in job record, require wallet param to verify ownership.

### H6. No Rate Limiting on Coach/Games Routes
- **Files**: All coach routes, games route
- **Description**: Unlike signing routes, coach/games have zero rate limiting. `/analyze` calls external LLM API (real cost).
- **Fix**: Add Upstash Ratelimit (already configured for signing routes).

## BROKEN FLOWS — Frontend

### F1. CoachLoading Polling Restarts Every 500ms (Never Completes)
- **Severity**: BROKEN
- **File**: `components/coach/coach-loading.tsx:41`, `app/arena/page.tsx:483-494`
- **Description**: `onReady`/`onFailed` are inline arrow functions (recreated every render). Listed as `useEffect` deps, so the polling interval restarts on every render (every 500ms when dots animate). Poll may never complete a 3s fetch cycle.
- **Reproduce**: Win game, ask coach, wait for async analysis. Loading spinner may stay forever.
- **Fix**: Wrap `onReady`/`onFailed` in `useCallback` in arena/page.tsx, or remove them from deps with a ref pattern.

### F2. Coach Paywall Buy Button Is a No-Op
- **Severity**: DEAD_END
- **File**: `app/arena/page.tsx:531`
- **Description**: `onBuy` is `() => { /* TODO */ setCoachPhase("idle") }`. User clicks buy, sheet closes silently, no purchase happens, no feedback.
- **Reproduce**: Exhaust 3 free credits, try to buy more.
- **Fix**: Show "Coming soon" message or disable button with explanation.

### F3. Session Storage Restores Stale Claim on New Game
- **Severity**: STATE_LEAK
- **File**: `app/arena/page.tsx:88-104`
- **Description**: Claim state restored from sessionStorage on mount, unconditionally. If user minted Victory A, returns to arena, wins game B, they see game A's success screen with stale data.
- **Reproduce**: Win, mint, return to arena, win again. Old victory data appears.
- **Fix**: Clear sessionStorage claim when starting a new game.

### F4. HUD Back Button Doesn't Reset Claim/Coach State
- **Severity**: STATE_LEAK
- **File**: `app/arena/page.tsx:423`
- **Description**: `game.reset` (used as onBack) only resets chess state. Claim phase, coach phase, sessionStorage not cleared.
- **Reproduce**: Win, start claiming, tap back arrow. State contaminated for next game.

### F5. Coach Welcome Overlay Has No Dismiss Path
- **Severity**: DEAD_END
- **File**: `app/arena/page.tsx:476-482`
- **Description**: Welcome overlay has no X button or tap-outside-to-dismiss. If wallet disconnects while showing, user is trapped.
- **Fix**: Add dismiss/close button.

## MEDIUM — Backend

### M1. Games Route: No GameRecord Validation
- **File**: `app/api/games/route.ts:10-28`
- **Description**: Accepts any object as `game`, stores in Redis. `moves` not validated as array. 100K-entry array = Redis memory exhaustion.
- **Fix**: Zod schema validation, limit moves array (max 500).

### M2. Unbounded RPC Log Scans (hall-of-fame, my-victories)
- **File**: `app/api/hall-of-fame/route.ts:44-54`, `app/api/my-victories/route.ts:53-64`
- **Description**: Scan from block 61.25M to latest in 50K chunks. Grows linearly with chain. No caching of intermediate results.
- **Fix**: Cache in Redis with short TTL, or use indexer/subgraph.

### M3. Error Messages Leak Internal Details
- **Files**: `leaderboard/route.ts:17`, `hall-of-fame/route.ts:91`, `coach/analyze/route.ts:159`
- **Description**: Raw `error.message` returned to client. RPC/LLM errors can contain URLs, API key fragments, rate limit info.
- **Fix**: Return generic error messages, log details server-side.

### M4. Free Credit Reset via Redis Eviction
- **File**: `app/api/coach/credits/route.ts:15-20`
- **Description**: `redis.set(credits, 3)` overwrites existing balance. If seeded key gets evicted, credits reset.
- **Fix**: Use `SETNX` for both seeded flag and credits.

### M5. Origin Check Bypassed Without Origin Header
- **File**: `lib/server/demo-signing.ts:63-66`
- **Description**: No Origin/Referer = check passes. MiniPay constraint, but `curl` also passes.
- **Status**: Known/accepted for MiniPay compatibility.
